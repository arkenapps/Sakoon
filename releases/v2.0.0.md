# Sakoon v2.0.0

**Sakoon · سكون** is now a standalone native Windows desktop application.

The same audio guard — muting system audio before each configured azaan
window, keeping the computer quiet through the window, and restoring the
previous audio state afterwards — now runs as a single self-contained
executable with its own app window. No browser tab, no localhost address, and
no open network ports of any kind.

## What changed

- **Native app window.** The interface renders inside the application itself
  instead of a browser tab pointed at `127.0.0.1`.
- **Zero network surface.** The loopback interface has been removed entirely.
  Sakoon opens no network ports; the app is designed to make no network
  connections at all.
- **Cleaner single-instance behaviour.** Launching Sakoon while it is already
  running simply brings the existing window to the front.
- **Close hides, quit quits.** Closing the window sends Sakoon to the system
  tray while the audio guard keeps running. Quit is available from the tray
  menu and restores audio on the way out.

## What did not change

- All timing, mute/unmute, verification, crash-recovery, snooze, sleep/wake, and
  Jumu'ah behaviour is identical to v1.0.0.
- The interface, including the Night, Pearl, Oasis, and Amber themes.
- Official uploaded tables remain the source of truth; calculated times remain
  clearly labelled **CALCULATED — UNOFFICIAL**.
- Data location: `%LOCALAPPDATA%\Sakoon`. Settings, uploaded tables, and logs
  from v1.0.0 are picked up automatically — no migration step.
- A visible start-with-Windows shortcut is still created automatically on
  first run in the per-user Startup folder, and can be turned off in
  Settings. No administrator access is required at any point.

## A note on the Windows warning

Sakoon is not code-signed, so Windows SmartScreen shows *"Windows protected
your PC"* the first time you run it. That warning reflects the fact that the
application is new and unrecognised, not anything found inside the file.
Choose **More info**, then **Run anyway**. `README-FIRST.txt` inside the zip
explains this, and every release publishes SHA-256 hashes if you would rather
verify the download before trusting it.

## Install / upgrade

1. Download `Sakoon-v2.0.0-Windows.zip` and `SHA256SUMS.txt`.
2. Verify the SHA-256 hash, then extract the zip.
3. If upgrading from v1.0.0: quit the running Sakoon from the tray first,
   then replace the old executable with the new one and run it. Your
   settings, tables, and logs carry over automatically.
4. On a fresh machine: run the executable, select the correct region, and
   upload the current official authority table when available.
5. Review the import preview before activation.

**Requirements:** Windows 10 or 11. The app window uses Microsoft Edge
WebView2, which ships with Windows 11 and current Windows 10; on the rare
machine without it, Windows offers a one-time free install on first launch.

## Important timing notice

Sakoon is not a religious authority. The relevant awqaf publication and local
mosque always take precedence. Calculated values are unofficial and Sakoon
should not be the only prayer reminder.

## Known limitations

- Windows only.
- New unsigned executables may trigger a Windows reputation warning. Download
  only from the official `arkenapps/Sakoon` release page and verify the
  published hash.
- Spreadsheet layouts with decorative or merged cells may require a cleaned
  copy.
- Audio behaviour can vary across Bluetooth, docks, vendor drivers, virtual
  audio devices, and endpoint changes.

## Privacy

Sakoon runs entirely on your machine with no open network ports. It does not
require an account and includes no advertisements or telemetry.

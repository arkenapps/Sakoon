Place the final release executable in this folder with the exact name:

Sakoon.exe

For v2.0.0 this is the single-file Wails build produced by build.bat in the
private Sakoon source folder. Its version resource must read 2.0.0.0 —
PUBLISH-V2-UPDATE.bat reads that resource and stops if it finds the old build,
because shipping yesterday's binary is the easiest mistake to make here.

The v1.0.0 executable and the old Sakoon_App.zip were removed from this folder
deliberately: v2.0.0 is distributed as one self-contained exe, and leaving a
stale binary next to a new release is how the wrong file gets uploaded.

SHA256SUMS.txt is regenerated by the publishing script on every run.

This placeholder is committed. Sakoon.exe and the generated SHA256SUMS.txt are
ignored by Git and uploaded only as GitHub Release assets.
